CSS has a lot of stuff that is fun to talk about. For this presentation we will focus on the layout aspects. Or to phrase it differently, we will focus on 'making things go where we want them to go'.

position

  • static
  • fixed
  • sticky
  • relative
  • absolute

static => 'I do nothing.'

      position: static;
      left: 30px; // does nothing
      right: 30px; // does nothing
This is a position static element.

The only use case where you would want to write position: static; is when you want to override a position value from another selector.

fixed => 'I don't have a daddy'

      position: fixed;
      bottom: 30px;
      right: 30px;
This is a position fixed element.

You want to use this when you want to display a container by taking it out of the normal flow of the webpage and controling where it will appear depending on the viewport.

sticky => 'I don't need javascript no mo''

      position: sticky;
      top: 0;
This is a position sticky element.

It's useful for navigation bars or table headers - something that you want to stay on the top of the screen when you start scrolling.

relative => 'I'm my own boss.'

      position: relative;
      left: 30px;
      top: 30px;
This is a position relative element.

It's relative to it's original position.

relative => 'Where's my daddy?'

      position: absolute;
      left: 30px;
      top: 30px;
This is a position absolute element.

Absolute element is relative to it's closes ancestor that has a position.

display

display property is used to determine the 'box' of the element

browser default stylesheet sets the display property for each of the valid html elements

There are more possible values, but here we will cover the most useful/used ones: inline, block, inline-block, flex, grid

display: block;

a block element uses up the full width of it's parent, and by default has the height adjusted to fit the content

div, p, h1-6 ... are by default block elements

This is a block element. A div.

One can apply width and height to a block element.

      height: 100px;
      width: 100px;
      background: red;

This is a paragraph, it goes to new line.

Ron is a born leader. this div goes to a new line because it's a block element, even though the div above has width constrained to 100px
you can apply padding and margin to a block element, and it will 'expand' and move the content around it to accommodate to the new dimensions

display: inline; is useful for formatting certain parts of the text, nothing else.

display: inline;

      height: 40px;
      width: 400px;
      background: red;
      padding: 50px;
This is a span element.

inline element ignores width, height, and margins work only on the x-axis

display: inline-block;

Similar example as with the inline, but here notice that the elements are redefining the rendering box of the parent properly, like the block element would + they are in the same line (like the inline element works).

Some span Another span

display: inline-block; is useful for creating navbars, or whenever you have a simple horizontal layout, and you want to stack blocks in a row

display: flex;

flex element works as a block element, but it determines how the immediate children will be distributed

currently flex doesn't support gutters, one needs to write the logic for the gutters

float

once upon a time there was only float

when float is applied to an element, it is taken out of the normal 'flow' of the page

That means that the dimensions of the element will not influence the parent dimensions.
